# 小红书素人稿件写作 Skill - 开发方案书

## Skill 定义

**一句话描述**：帮助小红书创作者批量生成素人风格推广文章的自动化工具，通过参考稿件库学习风格、人设定制内容调性、关键词验证确保SEO效果。

---

## 核心逻辑 (Core Logic)

### 完整流程

```
用户输入（交互式引导）
    │
    ├── 场景（用户指定，如：避孕场景、推迟月经场景）
    ├── 人设（用户描述，如：素人口吻、25岁女生）
    ├── 关键词（SEO词，如：内射）
    ├── 写作要求（字数、风格等）
    │
    ▼
┌─────────────────────────────────────┐
│         1. 飞书读取器                │
│    - 读取参考稿件库                 │
│    - 按场景筛选匹配的文章           │
│    - 如无飞书配置，使用Mock数据     │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         2. Prompt组装器              │
│    - 组装：场景+人设+参考+关键词     │
│    - 生成多个不同切入角度            │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         3. DeepSeek API生成          │
│    - 调用 deepseek-chat 模型         │
│    - 生成5篇（可配置）不同角度稿件   │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         4. 关键词验证器              │
│    - 正则匹配检测必须包含的关键词    │
│    - 未通过 → 标记需重新生成         │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         5. 雷同度检测器              │
│    - TF-IDF + 余弦相似度            │
│    - 阈值0.7标记、0.85需重写        │
│    - 5篇两两比对（10组）            │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│         6. 飞书写入器                │
│    - 创建新文档                     │
│    - 批量写入文章内容                │
└─────────────────────────────────────┘
```

### 交互式输入流程

1. 场景设置（必填，支持多场景逗号分隔）
2. 人设设定（必填，描述写稿人身份风格）
3. 关键词设置（必填，SEO关键词）
4. 额外要求（可选）
5. 文章数量（默认5篇，范围1-20）
6. 字数要求（默认800-1000字）
7. 风格要求（默认素人口吻）
8. 确认生成

## 行为准则与约束 (Constraints)

### 绝对不能做的事
- 不能预设固定场景类型（场景由用户输入决定）
- 不能假设只有避孕药这一个产品
- 不能在没有关键词验证的情况下输出（验证是必须关卡）

### 必须坚持的风格
- 素人口吻，第一人称
- 真实自然，不广告腔
- 每篇文章必须有独特切入角度
- 输出格式：标题 + 正文 + 标签

### 技术约束
- API优先DeepSeek（成本低效果好）
- 飞书集成作为可选项（不配置则用Mock）
- 所有路径使用相对路径或配置变量

## 当前版本 Prompt (System Prompt)

### 写作指令模板 (`prompts/write_article.md`)

```
你是一个小红书素人稿件写作者。

产品/场景：{scenes}
人设要求：{persona}
参考稿件：
{reference_articles}

必须包含的关键词：{keywords}
写作要求：{requirements}

请按照以下要求生成{count}篇不同的素人稿件，每篇使用不同的切入角度。

切入角度设计指南：
- 根据参考稿件库中的场景分布，设计{count}个不同的切入角度
- 确保角度之间有足够差异（叙事视角、情绪基调、使用时机等）
- 每篇需要有独特的开头和故事线

每篇稿件要求：
- 字数：{word_count}
- 包含关键词：{keywords}
- 风格：{style}
- 格式：标题 + 正文 + 相关标签

输出格式：
---
【稿件1】
标题：
正文：
标签：
---
【稿件2】
标题：
正文：
标签：
---
（以此类推）
```

## 关键迭代记录

### 有效的指令
- ✅ 场景由用户prompt指定，不预设
- ✅ 参考稿件库按场景动态筛选
- ✅ 切入角度基于参考库自动生成差异化
- ✅ 关键词验证作为强制关卡
- ✅ 雷同度检测确保多样性
- ✅ 交互式CLI引导输入（非命令行）
- ✅ readline支持终端编辑

### 被舍弃的方案
- ❌ 方案二（MCP工具链）：太复杂，方案一敏捷起步可迁移
- ❌ 预设避孕场景类型：场景应该动态由用户指定
- ❌ Anthropic/Claude API：改用DeepSeek（成本考虑）

### 技术选型变更
| 项目 | 原方案 | 现方案 | 原因 |
|------|--------|--------|------|
| LLM API | Claude | DeepSeek | 成本考虑 |
| 飞书集成 | 必需 | 可选（Mock模式） | 先跑通流程 |
